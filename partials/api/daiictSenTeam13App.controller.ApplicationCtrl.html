ApplicationCtrl
controller in module daiictSenTeam13App

Description

ApplicationCtrl

Controller of the daiictSenTeam13App

Methods

  • getData()

    Function called when /application page loads. The unique jobId of the job for which student wants to apply is passed to this view as a route parameter. The name of the job is retrieved based on this jobId and stored in a scope variable for displaying in the view.

    Returns
    undefined

    Does not return anything.

  • goTo(page)

    This function is used to redirect the user to either of the 4 pages, that are profile page, chatRooms page, jobs page or people page, based on what he/she has clicked on in the navigation bar displayed in the left of the screen. Note the following

    1. A professor is redirected to '/createChat' on clicking "Chat Rooms" in the nav bar whereas a student is redirected to '/chatRooms'.
    2. A professor is redirected to '/posting' on clicking 'Jobs' whereas a student is redirected to '/jobs'.
    Parameters
    ParamTypeDetails
    pagestring

    String that is passed according to the option clicked by the user in the navigation drawer displayed to the left of the screen.

    Returns
    undefined

    Does not return anything.

  • initMaterial()

    Initialises the Matrialise modules.

    Returns
    undefined

    Does not return anything.

  • logout()

    Ends the user's session and logs him out.

    Returns
    undefined

    Does not return anything.

  • resetValues()

    Resets all the input fields that were filled by the student who is attempting to apply for a job.

    Returns
    undefined

    Does not return anything.

  • submitApplication()

    This function is called when the student clicks on submit application. The validate function is called and if it returns true, an application for the given job is stored in the database, including informaton such as who applied contact email of the member who applied, the cover letter, date submitted on, jobId, job name etc. On error, a toast message is displayed prompting to try again later. Else, application is submitted successfully.

    Returns
    undefined

    Does not return anything.

  • validate()

    Validation function called when user tries to submit application for the job selected. Only if all fields are filled in, validate will return true.

    Returns
    boolean

    Whether data is valid or not
